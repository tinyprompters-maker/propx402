x402 Protocol

PropX402 uses the x402 open payment protocol developed by Coinbase. Instead of API keys or subscriptions, agents pay per request in USDC on Base mainnet 芒聙聰 autonomously, with no accounts or sessions.

How x402 works 1 脗路 Agent sends a normal HTTP POST to a paid endpoint
2 脗路 Server responds with 402 Payment Required + a payment payload
3 脗路 x402 client signs and broadcasts a USDC transfer on Base mainnet
4 脗路 Request is retried automatically with the payment proof header
5 脗路 Server verifies payment on-chain 芒聠聮 returns data

Partial data

PropX402 always returns a response even when individual sources fail. Failed sources return a note key instead of data. Only a geocoding failure produces a 500.

Rate limits

No hard rate limits 芒聙聰 x402 payment is the natural rate limiter. Bulk requests are capped at 20 addresses. Vercel serverless enforces a 30-second execution timeout.
